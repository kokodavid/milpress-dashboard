# Mobile App — Content Management Requirements

This document describes the content fields the mobile app reads from the
backend. The dashboard's **App Content** tab allows an admin to set and
update each of these fields. All URLs should point to publicly accessible
files hosted on Supabase Storage.

---

## Database Tables

### `app_content` — single-row global config

Always exactly one row (`id = 1`). Stores the video and thumbnail URLs.

| Column                       | Type        | Notes                        |
|------------------------------|-------------|------------------------------|
| `id`                         | int (PK)    | Always 1                     |
| `intro_video_url`            | text        | MP4                          |
| `intro_video_thumbnail_url`  | text        | JPG or PNG, 16:9 ratio       |
| `help_video_url`             | text        | MP4                          |
| `updated_at`                 | timestamptz | Auto-updated on every write  |

### `app_resources` — ordered list of downloadable resources

Any number of rows. The mobile app fetches these ordered by `display_order`
and renders each as a tappable download row in the Milpress Resources section.

| Column          | Type        | Notes                              |
|-----------------|-------------|------------------------------------|
| `id`            | uuid (PK)   | Generated by DB                    |
| `label`         | text        | Display name, e.g. "Alphabets Chart" |
| `file_url`      | text        | Supabase Storage URL — PDF or video file |
| `audio_url`     | text        | Supabase Storage URL — companion audio (MP3/M4A) |
| `type`          | text        | `'pdf'` or `'video'`               |
| `display_order` | int         | Ascending sort order in the app    |
| `created_at`    | timestamptz |                                    |

Two default rows are seeded by the migration:
- **Alphabets Chart** (pdf, order 1)
- **Mouth Sync Guide** (pdf, order 2)

## Content Fields

### 1. Intro Video URL
- **Table/column:** `app_content.intro_video_url`
- **Type:** String (URL)
- **File format:** MP4
- **Used in:** Home screen → Introduction tile → "What is Milpress" — tapping
  the thumbnail opens a video player dialog

---

### 2. Intro Video Thumbnail URL
- **Table/column:** `app_content.intro_video_thumbnail_url`
- **Type:** String (URL)
- **File format:** JPG or PNG, 16:9 ratio
- **Used in:** Home screen → Introduction tile → the still image shown over
  the video area before the user taps to play

---

### 3. Help Video URL
- **Table/column:** `app_content.help_video_url`
- **Type:** String (URL)
- **File format:** MP4
- **Used in:** Home screen header → "Need help?" button — tapping opens a
  video dialog

---

### 4. Downloadable Resources (list)
- **Table:** `app_resources`
- **Type:** Ordered list of `{ label, url, type, display_order }` rows
- **File formats:** PDF (type = `'pdf'`) or MP4 (type = `'video'`)
- **Used in:** Home screen → Introduction tile → "Milpress Resources" section
  — the app fetches all rows ordered by `display_order` and renders each as
  a tappable download/open row. Adding, removing, or reordering rows in the
  dashboard is immediately reflected in the app on next fetch.

## Summary

| Source                              | Field / Column                      | Format  |
|-------------------------------------|-------------------------------------|---------|
| `app_content`                       | `intro_video_url`                   | MP4     |
| `app_content`                       | `intro_video_thumbnail_url`         | JPG/PNG |
| `app_content`                       | `help_video_url`                    | MP4     |
| `app_resources` (one row per item)  | `label`, `file_url`, `audio_url`, `type`, `display_order` | PDF / MP4 + audio |

## Migration

Run `supabase/app_content_schema.sql` against your Supabase project to create
both tables, seed the default data, and configure RLS policies.
